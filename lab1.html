LABORATORIO NUMERO UNO

En este laboratorio se trabajaron consultas SQL en un sistema de servicios, aplicando comandos como DELETE para eliminar registros, UPDATE para actualizar datos, e INSERT INTO para agregar nueva información. También se usó WHERE para filtrar datos y se practicó el ordenamiento de resultados en forma ascendente y descendente. Esto permitió manipular y organizar datos de manera eficiente.

CONSULTAS

Ordenamiento (ASC/DESC):

ASC organiza los resultados en orden ascendente (por defecto) y DESC en orden descendente.

WHERE

Es una cláusula que se emplea para filtrar registros basados en una condición.

INSERT INTO

Se utiliza para agregar nuevos registros a una tabla, especificando los valores que se insertarán.

UPDATE

Sirve para actualizar datos en una tabla. También se combina con WHERE para indicar qué registros deben modificarse.

DELETE

Permite eliminar registros de una tabla. Se usa junto con la cláusula WHERE para especificar qué registros se deben eliminar. Estas consultas son fundamentales para la manipulación y organización de datos en bases de datos relacionales.

Tablas


Tabla Administrativos

  • Campos: ID (clave primaria), nombre, apellido, departamento, teléfono y correo electrónico.
  • Propósito: Almacenar los datos del personal administrativo de la institución.

Tabla Clases

  • Campos: ID de la clase (clave primaria), nombre de la clase, código, horario, ID del profesor y del salón.
  • Propósito: Describir las materias impartidas, vinculándolas con profesores y salones.

Tabla Estudiantes

  • Campos: ID del estudiante (clave primaria), nombre, apellido, fecha de nacimiento, teléfono, correo electrónico, carrera y semestre.
  • Propósito: Registrar los datos personales y académicos de los estudiantes.

Tabla Practicas

  • Campos: ID de la práctica (clave primaria), nombre, descripción, fechas de inicio y fin, ID del estudiante y de la clase.
  • Propósito: Documentar las actividades prácticas realizadas por los estudiantes.

    • Tabla Profesores

    • Campos: ID del profesor (clave primaria), nombre, apellido, especialidad, teléfono y correo electrónico.
    • Propósito: Registrar información sobre los docentes y sus áreas de especialización.

Tabla Salones

  • Campos: ID del salón (clave primaria), nombre o código, capacidad y ubicación.

    • Propósito: Identificar las aulas disponibles para la impartición de clases.


Esquema de Relaciones

Se deduce que se establecieron relaciones entre las tablas:


  • Los estudiantes están relacionados con las prácticas (campo ID_Estudiante)
  • Las prácticas están asociadas a clases (campo ID_Clase)
  • Las clases están vinculadas a profesores y salones mediante los campos ID_Profesor y ID_Salon

Proposito del laboratorio

Aprendizaje


  • Familiarizarse con el uso de phpMyAdmin para diseñar y gestionar bases de datos relacionales.
  • Diseño estructurado: Construir un sistema organizado que facilite el manejo de información de una institución educativa.
  • Validación: Asegurar que cada tabla cumple con las normativas de diseño de bases de datos, como evitar redundancias y establecer claves primarias para cada entidad.
